# Acceptance Criteria 템플릿

## 형식: Given / When / Then

```markdown
## Feature: [기능명]
**Story**: As a [역할], I want [목표], So that [비즈니스 가치]

### AC-1: [시나리오명 — Happy Path]
**Given** [사전 조건]
**When** [사용자 액션]
**Then** [기대 결과 + API 응답 코드]

### AC-2: [시나리오명 — Edge Case]
**Given** [사전 조건]
**When** [사용자 액션]
**Then** [기대 결과]

### AC-N: 비기능 요건
- [ ] 로딩 상태 표시 (스켈레톤 또는 스피너)
- [ ] 에러 시 toast 메시지 표시
- [ ] 빈 결과 시 EmptyState 표시
```

## 예시 1 — 채용공고 지원

**Story**: 구직자로서 채용공고에 지원하고 싶다. 내 이력서를 선택해서 제출할 수 있어야 한다.

### AC-1: 이력서 선택 후 지원 성공
**Given** 로그인된 구직자이고, 이력서가 1개 이상 있고, 공고 상세 페이지에 있다
**When** "지원하기" 버튼 클릭 → 이력서 선택 모달 열림 → 이력서 선택 → "지원" 버튼 클릭
**Then** `POST /api/applications` 201 반환, 성공 toast 표시, 버튼이 "지원완료"로 변경

### AC-2: 이력서 없을 때 지원 시도
**Given** 로그인된 구직자이고, 이력서가 0개다
**When** "지원하기" 버튼 클릭
**Then** "이력서를 먼저 작성해주세요" 안내 모달 표시, 이력서 작성 페이지 링크 제공

### AC-3: 미로그인 상태에서 지원 시도
**Given** 미로그인 상태
**When** "지원하기" 버튼 클릭
**Then** 로그인 페이지로 리다이렉트 (return URL 포함)

### AC-4: 중복 지원 시도
**Given** 이미 지원한 공고
**When** "지원하기" 버튼 클릭
**Then** `POST /api/applications` 409 반환, "이미 지원한 공고입니다" toast

### AC-5: 비기능 요건
- [ ] 지원 중 버튼에 로딩 스피너 표시
- [ ] API 미구현 동안 disabled 처리 + "준비 중" 안내
- [ ] 이력서 목록 로딩 중 스켈레톤 표시

## 예시 2 — 채용공고 목록 조회

**Story**: 구직자로서 조건에 맞는 채용공고를 탐색하고 싶다.

### AC-1: 공고 목록 조회
**Given** `/jobs` 페이지 접속 (미로그인 가능)
**When** 페이지 로드
**Then** `GET /api/posts/company/list` 호출, 공고 카드 그리드 표시

### AC-2: 고용형태 필터
**Given** 공고 목록 페이지
**When** "정규직" 필터 선택
**Then** employment_type = "정규직"인 공고만 표시 (클라이언트 필터링)

### AC-3: 급여 정렬
**Given** 공고 목록 페이지
**When** "급여 높은 순" 정렬 선택
**Then** salary 내림차순으로 공고 재정렬

### AC-4: 검색
**Given** 공고 목록 페이지
**When** 검색창에 키워드 입력
**Then** title 또는 work_location에 키워드가 포함된 공고만 표시

### AC-5: 빈 결과
**Given** 필터 결과가 0개
**When** 필터 적용
**Then** EmptyState 컴포넌트 표시 ("조건에 맞는 공고가 없습니다")

## 예시 3 — 기업 프로필 수정

**Story**: 기업 담당자로서 회사 정보를 최신 상태로 유지하고 싶다.

### AC-1: 프로필 수정 성공
**Given** 로그인된 기업 담당자, `/company/profile/edit` 페이지
**When** 필드 수정 후 저장
**Then** `PATCH /api/company-profile` 200 반환, 성공 toast

### AC-2: 필수 필드 누락
**Given** 프로필 편집 폼
**When** 필수 필드(회사명 등) 비워두고 저장
**Then** 클라이언트 Zod 검증 실패, 필드 하단에 에러 메시지 표시

### AC-3: 비기능 요건
- [ ] 저장 중 버튼 로딩 상태
- [ ] sticky 저장 버튼 (스크롤해도 보임)
- [ ] 페이지 나갈 때 unsaved changes 경고

## AC 작성 체크리스트

작성 완료 전 확인:

- [ ] 모든 Happy Path AC가 있는가?
- [ ] 인증 관련 Edge Case가 있는가? (미로그인, 권한 없음)
- [ ] API 미구현 시 대응 방안이 있는가?
- [ ] 로딩/에러/빈 상태 AC가 있는가?
- [ ] 각 AC에 API 응답 코드가 명시되어 있는가?
- [ ] 비기능 요건(접근성, 성능)이 있는가?
